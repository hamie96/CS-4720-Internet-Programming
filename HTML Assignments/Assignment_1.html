Requirements

Create Class Collector

Create a class named Collector in a file named collector.py. Itâ€™s necessary to specify all this so that the unit testing will work properly.

The class collects numerical values. Values are added one at a time by the add method described below. Various methods will return statistical values based on the values collected.

Here are the methods required. You may wish to add other methods. For example an __init__ method is probably necessary.

add(self, x) Add the value x to the collection
count(self) Return the number of values collected so far
sum(self) Return the sum of the values collected so far
sum_squares(self) Return the sum of the squares of the values collected so far
average(self) Return the average of the values collected so far
variance(self) Return the variance of the values collected so far
standard_deviation(self) Return the standard deviation of the values collected so far

The main requirement is that the six methods other than add provide results correct for the values collected. You may decide to store the collected values in a list and compute from that list. There are alternatives that do not involve storing the list but, rather, a few numeric variables. This is your choice.

Demonstration Script

In the same folder with collector.py, create a script tryCollector.py. This script should create a Collector, add a few values to it and display the various statistics available, suitably labeled.

You may wish to figure out what the statistics should be, but that is not required. The unit test, next section, is designed to look at those questions. The objective here is to properly use the class.

Testing

The Python script test1.py is a unit test. This runs a single test case that should succeed. In PyCharm, you can see the test results in views on the lower part of the window. The screen shot to the left is an example.

Place the file test1.py in the same folder with your file collector.py. It should run properly from there.

Note: if you have to make changes to test1.py to make it run, be sure to check with your instructor! A version of this test will be used in evaluating assignments. The test is presumed correct until problems are noted.

test1A.py is an additional test that checks out possible errors that would not show up in test1.py. In particular, this test uses two Controller objects in parallel. They should not interfere with each other.

Submit
