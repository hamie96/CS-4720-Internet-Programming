Objectives

  • Serve a report through a web server
  • Compare the performance of choices in designing the protocol

Requirements

Start with a copy of Assignment 4.

Create a server that serves the report as a web page. Use a table to make sure the report is properly formatted.

  • The web page will contain the report

    • The web page should reference the style sheet below
      • Change the two background colors, but make sure the report is readable
      • Use the grid class on the table you create for the report
      • Add some padding to the cells in the table by adding a property and value to the table.grid rule (do not do this by adding attributes to the tags, it must happen just by using the grid class)

Stylesheet to use

html {
    background-color: aquamarine;
}

body {
    background-color: skyblue;
    width: 50%;
    margin-left: 25%;
}


table.grid, table.grid th, table.grid td {
    border-style: solid;
    border-collapse: collapse;
    border-width: 3px;
}

Bonus (10 points)

Do this only if you have completed the Assignment 4 and the required part of Assignment 5.

  • Create a service, add to db_access that will provide all the data needed in a single request
    • Create a new function to add to db_access. The idea is that one call of this function will provide the data needed for the report.
    • Add to the document you created for Assignment 4. Create a new section in the document and describe what data is returned by the request, how it is organized and how it will be used.
  • Copy the first REST server and add new service
    • The new server will be called server2
    • Extend the protocol (and document that)
    • Add the service to the new server
    • Use port 21212 for this server
  • Re-implement the web report to use this service, call it web_server2
  • Compare performance
    • Time the new report program and compare that to the old one
    • Put this in the document

Evaluation for Grade

  • The web_server client will be accessed with a browser
  • Server and client code will be examined for appropriateness and readability
