Objectives

  • Use a form based application to allow the user to investigate details about the measurement data

Requirements

  1. There will be a web server at port 8000 that the user will contact to run the application
    1. There will be a REST server providing information to the web server.
      1. That will be at port 21212
      2. Use the REST server from the last assignments, augmenting it as necessary
  2. The user will navigate among three different pages, and possibly error pages
    1. Area select page, the start page
    2. Area information page
    3. Location information page
  3. The Area select page will present a list of areas, by name, for the user to select.
    1. Selecting an area and clicking the button will take the user to the Area information page
  4. The Area information page will present information about the area.
    1. At the top will be general information: id, name, latitude and longitude
    2. The average of all measurements for that area will be displayed at the top, if available. Otherwise, an appropriate message will be displayed in place of the value
    3. If the area has locations, they are displayed in a table below
    4. If the area has no locations, a message is displayed in place of the table. NOTE: do not display an empty table here!
    5. Each row in the location table will have a button in the first column. Clicking the button will take the user to the Location information page for that location
  5. The Location information page will display information about the selected location
    1. At the top will be general information: id, name, altitude, name of the area to which this location belongs, count of the measurements for this location, average of the measurements for this location (if any)
      1. The name of the area should be made into a link (a tag) that will lead to the page for that area. You will have to manually include a query in the URL used in the link.
    2. Below that will be a table of measurements, if there are any. Display the measurement id and the value of each measurement.
      1. If there are no measurements for that location, display a suitable message. Do not display an empty table! Note that the North Pass location has no measurements, so be sure to test that.
  6. Here are errors you should deal with
    1. Receiving no id from the user when requesting the area information page or the location information page
      1. Do not attempt to display the page you would display if there were no error
      2. Display a special page that explains the problem to the user (you donâ€™t have to provide a really detailed explanation)
    2. Receiving a non-existing id from the user. Treat this in a similar way to the previous case
  7. Use the following paths for pages:
    1. / for the area select page
    2. /area-info for the area information page
    3. /location-info for the location information page

Evaluation for Grade

  • The web server will be accessed with a browser, using it in the expected manner
  • Specially crafted pages will be used to access the web server
    • These will test the error handling
    • It is very important that you use the ports and paths specified above
  • Server code will be examined for appropriateness and readability
