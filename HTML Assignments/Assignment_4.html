Objectives

  • Design a REST protocol
  • Implement a REST protocol
  • Use a REST service to create a report
  • Compare the performance of choices in designing the protocol

Requirements

Start with services provided by db_utility and db_access from Assignment 2. Put them in a new Python project directory (or package, if you wish).

  • Design an appropriate mapping of paths to services
    • Use the example in REST design as a guide to your design
    • Use the tables in REST design as a model to describe requests and parameters
    • Create a document to describe your design and include that in the project directory.
  • Implement a server based on the bottle package
    • This will implement the protocol you just designed
    • Name the server server1
    • Use port 21212 for this server
  • Create a client that will create a report similar to the one in Assignment 2 but depending on the REST
    • Name the script report1
    • Client uses the requests package to access the REST server
  • Measure performance
    • How long does it take to produce the report from when the program starts running?
    • May wish to extrapolate from partial results if the program is very slow
    • Report this in a new section of the document you created for the design

Evaluation for Grade

  • The client will be run
  • Server will be accessed from a browser
  • The document will be evaluated for the required content and for good communication
  • Server and client code will be examined for appropriateness and readability

Submitting the Assignment

Zip up your project and submit it through the appropriate D2L folder.
